## ðŸŽ¯ Key Design Decisions

### 1. Singleton MqttManager
- **Why**: Single MQTT connection per app
- **Benefit**: Shared state across ViewModels
- **Pattern**: Thread-safe double-checked locking

### 2. Repository Pattern
- **Why**: Separation of data layer from UI
- **Benefit**: Easy to swap MQTT implementation
- **Pattern**: Abstract data operations

### 3. StateFlow for State
- **Why**: Lifecycle-aware, backpressure-safe
- **Benefit**: Automatic UI updates
- **Pattern**: Reactive programming

### 4. MVVM Architecture
- **Why**: Separation of concerns
- **Benefit**: Testable, maintainable
- **Pattern**: Industry standard for Android

### 5. Compose Navigation
- **Why**: Type-safe navigation
- **Benefit**: Deep linking support
- **Pattern**: Declarative navigation
